# Publish — Кодомандри

Послідовність публікації нових днів завдань.

---

## Крок 1 — Валідація (check)

```bash
node scripts/validate-tasks.js --check
```

Вивести підсумок: скільки ✅ / ⚠️ / ❌. Якщо є ⚠️ або ❌ — перейти до кроку 2.

---

## Крок 2 — Автофікс координат

```bash
node scripts/validate-tasks.js --fix
```

Після фіксу — повторити крок 1 щоб переконатись що всі ✅.

---

## Крок 3 — Commit + Push

```bash
git add data/tasks.json
git commit -m "fix: correct task coordinates for days X–Y"
git push
```

---

## Крок 4 — Тестування в браузері (ручне)

Вивести список днів що були виправлені, попросити юзера вручну перевірити кожен день:
```
https://phase0-five.vercel.app/app/task.html?uid=<uid>&task=task_dX_m1
```

Запитати: **"Перевірив? Є проблеми?"**

---

## Крок 5 — Bug Tracker (якщо є проблеми)

Якщо юзер каже що є проблеми → запустити скіл `bug-tracker` (Операція 1: додати баг).

---

## Крок 6 — Підсумок

Вивести:
```
✅ Днів виправлено: X
✅ Координат пофіксовано: Y
🚀 Vercel деплоїть автоматично
📋 Нові баги: Z (або "немає")
```
