# Bug Tracker — Кодомандри

## Ціль

Sprint Board у Notion: `71f5f39e-6bea-4e24-9d2c-222afb4ef8b3`
URL: https://www.notion.so/40a4258990e84e2ea9c34ffacc1fdcd1

## Операція 1: Додати баг

Створи запис через Notion MCP з полями:

| Поле | Значення |
|------|----------|
| `Назва` | `🐛 [короткий опис]` |
| `Тип` | `Task` |
| `Статус` | `To Do` |
| `Пріоритет` | див. таблицю нижче |
| `Спринт` | поточний активний спринт |
| `Acceptance Criteria` | файл/компонент + умови відтворення |

### Визначення пріоритету

| Пріоритет | Коли |
|-----------|------|
| `🔴 Критичний` | дитина не може виконати завдання / функціонал зламаний |
| `🟡 Середній` | є обхідний шлях, але заважає роботі |
| `🟢 Низький` | косметика, не впливає на навчання |

**Евристика для Кодомандри:** якщо баг у `lesson-engine`, `server.js`, перевірці блоків або TTS — швидше за все 🔴. Якщо UI/стилі/тексти — 🟢.

---

## Операція 2: Позначити як виправлений

Знайди запис за назвою (`🐛 ...`) і постав `Статус: Done`.

---

## Операція 3: Список відкритих багів

Знайди всі записи де `Тип = Task` і назва починається з `🐛` і `Статус ≠ Done`. Виведи у форматі:
#N | 🔴/🟡/🟢 | Назва | Спринт
