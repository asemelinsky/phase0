# Skill: generate-task

## Призначення
Генерація JSON-конфігурацій завдань для платформи Scratch Learning.
Завдання для дітей 6–9 років. Мова — українська.

## JSON-формат завдання (обов'язкові поля)

```json
{
  "id": "task_001",
  "level": "novice",
  "title": "Котик і зірка",
  "character": "cat",
  "description": "Допоможи котику дістатися до зірки!",
  "objective": "Склади програму з 3 кроків щоб котик пройшов до зірки",
  "availableBlocks": ["move_right", "move_up", "jump"],
  "solution": ["move_right", "move_right", "jump"],
  "hints": [
    "Котику потрібно спочатку рухатися вправо",
    "Після двох кроків вправо — стрибок!",
    "Спробуй: вправо → вправо → стрибок"
  ],
  "canvas": {
    "character": "cat",
    "startX": 50,
    "startY": 200,
    "target": { "type": "star", "x": 250, "y": 150 },
    "obstacles": []
  },
  "audio": {
    "intro": "Привіт! Допоможи мені дістатися до зірочки!",
    "success": "Ура! Ти зробив це! Ти справжній програміст!",
    "hint": "Підказка: спробуй рухатися вправо спочатку"
  }
}
```

## Рівні складності

### Новачок (novice) — 1–3 блоки в розв'язку
- Тільки базові рухи: move_right, move_left, move_up, move_down
- Без циклів і умов
- Один персонаж, проста ціль
- Завдання 001–010

### Середнячок (intermediate) — 4–8 блоків, можуть бути цикли
- Додаються: repeat(N), if_obstacle
- 2–3 кроки до цілі з перешкодами
- Завдання 011–020

### Досвідчений (advanced) — комбінації, кілька правильних відповідей
- Оптимізація: "зроби це за мінімум кроків"
- Комбінації: цикл + умова
- Завдання 021–030

## Персонажі (character field)
cat, dog, parrot, snail, frog, rabbit, robot, bee, turtle

## Доступні блоки (availableBlocks)
Базові рухи: `move_right`, `move_left`, `move_up`, `move_down`, `jump`
Цикли: `repeat_2`, `repeat_3`, `repeat_5`, `always`
Події (Pro): `event_flag`, `event_click`
Умови (Pro): `if_obstacle`, `if_touching_sign`, `if_near_creeper`, `if_wall`, `if_dragon_fire`
Умови-гілки (Pro): `if_else_safe`, `if_else_bridge_ok`
Дії (Pro): `say_alert`, `repair_bridge`, `hit_hammer`, `use_water`, `strike_sword`

**ВАЖЛИВО ДЛЯ PRO-РІВНІВ:** 
1. `solution` **ПОВИНЕН** починатися з події (напр. `"event_flag"`).
2. Завжди включай відповідну подію в `availableBlocks`.
3. У JSON-структурі Blocky, якщо блок містить інші блоки, вони кладуться в масив, який іде НАСТУПНИМ елементом після самого блоку. Наприклад, подія з циклом всередині:
```json
"solution": [
    "event_flag",
    [
         "always",
         [
             "move_right",
             "if_wall",
             [ "jump" ]
         ]
    ]
]
```

## Алгоритм генерації пачки з 10 завдань

1. Прочитай поточний data/tasks.json щоб знати яка нумерація вже є
2. Визнач рівень (novice/intermediate/advanced) з запиту користувача
3. Генеруй 10 завдань з наростаючою складністю в межах рівня
4. Кожне завдання — новий персонаж (не повторювати підряд)
5. Перевір що solution відповідає availableBlocks
6. Перевір що кожне завдання педагогічно правильне (нова концепція = +1 блок)
7. Збережи ТІЛЬКИ нові завдання (append до data/tasks.json, не перезаписуй)

## Педагогічні правила
- Нова концепція вводиться по одній (не два нових блоки одночасно)
- Кожне завдання має 3 підказки: загальна → конкретніша → майже рішення
- Аудіо-фрази: короткі, радісні, для 6–9 років
- НЕ використовувати: "програма", "алгоритм", "код" — тільки "команди", "кроки", "підказки"
