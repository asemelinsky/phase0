## Мета
Взяти запис з Notion DB навчальних анімацій (ідентифікований по полю `ID`), згенерувати файли анімації і додати її на preview-сторінку.

## Кроки

### 1. Отримати ID анімації
Якщо користувач не вказав ID — запитай: «Вкажи ID анімації (наприклад `anim_drag_to_workspace`)»

### 2. Знайти запис в Notion
Зроби пошук в Notion DB `collection://7ab489b2-db93-43f5-8e90-a1350567b8e2` по ID.
Отримай поля: **Назва**, **ID**, **Опис дії**, **Де показується**, **Тригер**, **Показувати**.

Якщо запис не знайдено — повідом і зупинись.

### 3. Згенерувати файли анімації

Створи два файли в `app/animations/`:

**`{id}.css`** — CSS-стилі та keyframes.
- Базуй анімацію на описі з поля "Опис дії"
- Для стрілок/рухів використовуй `@keyframes` з `transform: translate`
- Для підсвічування кнопок — `@keyframes` з `box-shadow` пульсацією
- Клас контейнера: `.{id}-overlay` (fixed, fullscreen, z-index 2000)
- Клас dismiss-кнопки: `.{id}-dismiss`
- Завжди використовуй CSS-змінні: `--primary: #6366f1`, `--accent: #10b981`

**`{id}.js`** — логіка анімації.
Структура файлу:
```js
// {id} — {Назва}
AnimEngine.register('{id}', {
    meta: {
        id: '{id}',
        name: '{Назва}',
        description: '{Опис дії}',
        where: [{Де показується як масив}],
        trigger: '{Тригер}',
        showRule: '{Показувати}',
    },

    shouldShow(ctx) {
        // якщо "Тільки один раз" — перевіряти localStorage
        // якщо "Щоразу" — завжди true
        // якщо "До певної дії" — перевіряти ctx.done
    },

    show(ctx) {
        _ensure{PascalId}DOM();
        // показати overlay, запустити анімацію
    }
});

function _ensure{PascalId}DOM() {
    if (document.getElementById('{id}-overlay')) return;
    // inject HTML template
}

function _{id}Dismiss() {
    const el = document.getElementById('{id}-overlay');
    if (el) el.remove();
}
```

### 4. Логіка анімації залежно від "Опис дії"

| Тип опису | Що генерувати |
|---|---|
| "стрілка", "перетягни", "рухається" | Анімована рука/стрілка що рухається по екрану |
| "підсвічуємо кнопку", "мигаємо" | Overlay з highlight-ring навколо потрібного елемента |
| "з'єднай", "клацають" | Два блоки що рухаються назустріч і з'єднуються |
| "zoom", "збільшення" | Демонстрація +/- кнопок з анімацією масштабу |
| "показуємо варіанти" | Список кроків з почерговою підсвіткою |

Анімація має:
- **Показуватись поверх інтерфейсу** (fixed overlay з напівпрозорим фоном або без нього)
- **Мати кнопку закриття** "Зрозумів! 👍" або "OK"
- **Автоматично зникати** через 6–8 сек якщо юзер не закрив
- **Не блокувати весь екран** — overlay тільки там де потрібно показати дію

### 5. Додати в preview.html

Відкрий `app/animations/preview.html` і зроби такі зміни:

1. В `<head>` додай `<link rel="stylesheet" href="{id}.css">` поряд з іншими
2. Перед `</body>` додай `<script src="{id}.js"></script>` поряд з іншими
3. В масив `PREVIEWS` додай об'єкт:

```js
{
    id: '{id}',
    name: '{Назва}',
    desc: '{Опис дії}',
    tags: [{теги з Де показується + Тригер + Показувати}],
    run() {
        // скинути localStorage якщо потрібно
        AnimEngine.trigger('{id}', { /* mock ctx */ });
    },
    reset() {
        // видалити overlay або скинути стан
        const el = document.getElementById('{id}-overlay');
        if (el) el.remove();
    }
},
```

### 6. Перевірити результат

Після генерації:
- Виведи список створених файлів
- Скажи: `Відкрий app/animations/preview.html у браузері і натисни ▶ Показати для «{Назва}»`

### 7. Не робити

- ❌ Не модифікувати `task.html`, `task.js`, `progress.html` — це окремий крок підключення
- ❌ Не запускати сервер
- ❌ Не комітити — лише повідомити що файли готові

### 8. Блоки Blockly в анімаціях

Якщо нова анімація потребує відображення Blockly-блоку (SVG) — **не генерувати нові визначення**.
Використовувати `BLOCK_DEFS` з `app/animations/intro_overlay.js`:
- Там є `label`, `hex`, `desc` для всіх наявних блоків
- Якщо потрібного блоку немає в `BLOCK_DEFS` — додати його туди (єдине місце)
- `intro_overlay` вже підключений до `task.html` і викликається автоматично через `AnimEngine` коли в завданні є поле `intro_block: { "id": "..." }`
