# Skill: test-runner

**Тригери:** "запусти тести", "run tests", "перевір тести", "npm test", "що зламалось"

## Що робить

Запускає Jest тести і звітує про результат.

## Кроки

1. Виконай `npm test` в кореневій директорії проєкту
2. Якщо всі тести пройшли — повідом `✅ N тестів пройшли`
3. Якщо є помилки — покажи:
   - Список файлів з помилками
   - Назву кожного тесту що упав
   - Причину (перший рядок помилки)
4. Не фіксуй помилки самостійно — тільки звітуй

## Корисні команди

```bash
npm test                          # всі тести
npm test -- --testNamePattern="flattenSolution"   # конкретний тест
npm test -- --verbose             # детальний вивід
```

## Файли тестів

- `tests/engine.test.js` — Unit тести ігрового рушія (lib/engine.js)

## CI

GitHub Actions автоматично запускає `npm test` на кожен push у main/master.
Файл: `.github/workflows/test.yml`
