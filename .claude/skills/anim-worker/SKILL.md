## Крок 1 — Завантажити список

Отримати всі картки зі статусом `В Роботі` через Notion MCP.

Вивести у чат:
```
📋 Анімації в роботі:
1. [ID] — [Назва] — [Де показується] — [Тригер]
2. ...
```

Запитати: **"Яку анімацію беремо? (номер або ID)"**

> Якщо користувач одразу вказав ID (напр. `anim_drag_to_workspace`) — пропустити цей крок.

---

## Крок 2 — Читання картки і коментарів

Завантажити картку через `notion-fetch` по URL.

Прочитати:
- Поля: `Опис дії`, `Де показується`, `Показувати`, `Тригер`
- Тіло сторінки (якщо є)
- **Обов'язково** всі коментарі через `notion-get-comments`

**Фільтрація коментарів:**
- Коментарі з префіксом `🤖 Кладу |` — мої попередні звіти
- Коментарі **без** цього префіксу — зауваження від власника → **пріоритет над Опис дії**

**Правило:**
- Є коментарі від власника після останнього `🤖 Кладу |` → виконувати їх
- Немає коментарів → виконувати `Опис дії` картки

Вивести у чат: що саме треба зробити (1-3 пункти).

> Якщо в коментарях є "обговорити", "уточнити" — розпочати діалог замість реалізації.

## Крок 3 — Виконати

### 3а. Доопрацювання існуючої анімації

Знайти файли в `app/animations/`, читати код, вносити зміни.

### 3б. Створення нової анімації

#### Файли

Створити два файли в `app/animations/`:

**`{id}.css`**
- Клас контейнера: `.{id}-overlay` (fixed, fullscreen, z-index 2000)
- Клас dismiss-кнопки: `.{id}-dismiss`
- CSS-змінні: `--primary: #6366f1`, `--accent: #10b981`
- Тип анімації залежно від опису:

| Опис містить | Що генерувати |
|---|---|
| "стрілка", "перетягни", "рухається" | `@keyframes` з `transform: translate` |
| "підсвічуємо", "мигаємо" | `@keyframes` з `box-shadow` пульсацією |
| "з'єднай", "клацають" | два елементи рухаються назустріч |
| "zoom", "збільшення" | анімація масштабу `scale` |
| "показуємо варіанти" | почергова підсвітка кроків |

**`{id}.js`**
```js
// {id} — {Назва}
AnimEngine.register('{id}', {
    meta: {
        id: '{id}',
        name: '{Назва}',
        description: '{Опис дії}',
        where: [{Де показується}],
        trigger: '{Тригер}',
        showRule: '{Показувати}',
    },

    shouldShow(ctx) {
        // "Тільки один раз" → перевіряти localStorage
        // "Щоразу" → завжди true
    },

    show(ctx) {
        _ensure{PascalId}DOM();
        // показати overlay, запустити анімацію
    }
});

function _ensure{PascalId}DOM() {
    if (document.getElementById('{id}-overlay')) return;
    // inject HTML
}

function _{id}Dismiss() {
    const el = document.getElementById('{id}-overlay');
    if (el) el.remove();
}
```

**Вимоги до анімації:**
- Поверх інтерфейсу (fixed overlay)
- Кнопка закриття "Зрозумів! 👍" або "OK"
- Автоматично зникає через 6–8 сек
- Озвучення через `AppTTS.speak(text)` — якщо є текст для учня

#### Підключення в preview.html

1. `<head>` → `<link rel="stylesheet" href="{id}.css">`
2. Перед `</body>` → `<script src="{id}.js"></script>`
3. В масив `PREVIEWS` додати:

```js
{
    id: '{id}',
    name: '{Назва}',
    desc: '{Опис дії}',
    tags: [{Де показується, Тригер, Показувати}],
    run()   { AnimEngine.trigger('{id}', { /* mock ctx */ }); },
    reset() { const el = document.getElementById('{id}-overlay'); if (el) el.remove(); }
},
```

#### Підключення в progress.html / task.html

Тільки якщо анімація вже готова до показу учням. До того — **не чіпати**.

#### Блоки Blockly в анімаціях

Якщо потрібен SVG Blockly-блоку — використовувати `BLOCK_DEFS` з `app/animations/intro_overlay.js`. Не генерувати нові визначення в інших файлах. Якщо блоку немає в `BLOCK_DEFS` — додати туди.

#### Тіло Notion-картки

Після створення файлів — заповнити тіло сторінки через `notion-update-page` за структурою:
`Що робить / Файли / Логіка / localStorage / Запуск вручну`
