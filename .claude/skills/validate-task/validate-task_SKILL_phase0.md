# Skill: validate-task (v2)

## Призначення
Перевірка і автовиправлення завдань у `data/tasks.json`.
Гарантує відповідність між `solution` (маршрут героя) і `target` (координати цілі).

## Коли використовувати
- Додав нове завдання → запусти `--check` перед збереженням
- Після редагування `solution` або координат → запусти `--fix`
- Підозра що герой не доходить до цілі → запусти `--check`

---

## Крок 1: Запустити скрипт-валідатор

```bash
# Перевірка (без змін у файлі):
node scripts/validate-tasks.js --check

# Або через npm:
npm run validate
```

Читай звіт: `✅` — OK, `⚠️ КООРДИНАТИ` — розбіжність маршруту і цілі, `❌` — структурна помилка.

---

## Крок 2: Автовиправлення координат (якщо є ⚠️)

```bash
# Перераховує target.x / target.y під фактичний маршрут з solution:
node scripts/validate-tasks.js --fix

# Або через npm:
npm run fix
```

Режим `--fix` **не змінює** `solution`, `startX/Y`, чи будь-що інше — тільки `target.x` і `target.y`.

---

## Крок 3: Ручна валідація (чеклист)

Лише для помилок які скрипт НЕ виправляє автоматично:

### Структура JSON
- [ ] Всі обов'язкові поля: `id`, `level`, `title`, `character`, `description`, `objective`, `availableBlocks`, `solution`, `hints`, `canvas`, `audio`
- [ ] `id` унікальний, `level` одне з `novice/intermediate/advanced`
- [ ] `character` одне з: `cat, dog, parrot, snail, frog, rabbit, robot, bee, turtle`
- [ ] `hints` — рівно 3 елементи
- [ ] `audio` — три ключі: `intro`, `success`, `hint`

### Педагогіка
- [ ] Мова — тільки українська (title, description, hints, audio)
- [ ] `hints` від загального до конкретного
- [ ] `novice` рівень: 1–3 блоки в `solution`
- [ ] Всі блоки `solution` є в `availableBlocks`

### Унікальність
- [ ] `title` не повторюється
- [ ] Персонажі не йдуть підряд у послідовності завдань

---

## Як скрипт симулює рух

Кожен блок у `solution` збільшує позицію на **50px** (розмір клітинки сцени):

| Блок | Дія |
|------|-----|
| `move_right` | x += 50 |
| `move_left`  | x -= 50 |
| `move_up`    | y -= 50 |
| `move_down`  | y += 50 |
| `jump`       | не змінює клітинку |
| `repeat_3/5` | розгортає вміст N разів |
| `always`     | розгортає вміст 20 разів |

## Дії при помилках

| Помилка | Дія |
|---------|-----|
| `⚠️ КООРДИНАТИ` | Запусти `--fix` |
| `❌ Відсутнє поле` | Додати поле вручну |
| `❌ Дублікат id` | Змінити id вручну |
| `❌ Невідомий character` | Виправити назву |
| Не українська мова | Перекласти вручну |
