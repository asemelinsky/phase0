# План робочої сесії: 7 березня 2026 року

Цей план фокусується на переході від поодиноких ігрових завдань до підтримки кількох персонажів та покращенні мобільної версії гри.

## Пріоритетні цілі

### 1. Реалізація підтримки декількох персонажів (Multi-Character Support)
Згідно з погодженим [планом впровадження](file:///D:/aiscratch/claude/phase0/docs/Multiplayer%20Implementation%20Plan.md):
- [ ] **Створення CharacterManager**: Розробка логіки для керування станами кількох героїв одночасно.
- [ ] **Інтерфейс перемикання**: Додавання карток героїв над робочою областю Blockly.
- [ ] **Незалежні простори Blockly**: Реалізація збереження коду для кожного персонажа окремо при перемиканні.
- [ ] **Паралельне виконання**: Оновлення рушія анімації для одночасного руху всіх героїв, що мають код.

### 2. Мобільна адаптація (Responsive Design)
- [ ] **Перевірка поточного стану**: Тестування інтерфейсу на різних розмірах екранів через браузерні інструменти.
- [ ] **CSS Медіа-запити**: Додавання стилів для смартфонів (`max-width: 768px`) та планшетів.
- [ ] **Оптимізація Blockly**: Налаштування автоматичного масштабування блоків для зручного перетягування пальцями.

### 3. Підготовка до Спринту 3 (Supabase Integration)
- [ ] **Створення схеми БД**: Опис таблиць для користувачів та їхнього прогресу.
- [ ] **Налаштування підключення**: Створення допоміжного модуля для роботи з Supabase API.

## Розклад сесії (орієнтовно)
- **09:00 - 11:00**: Робота над логікою `CharacterManager` та UI картками.
- **11:30 - 13:30**: Рефакторинг рушія виконання та тестування паралельних рухів.
- **14:30 - 17:00**: Верстка адаптивного дизайну та фінальне тестування на мобільних телефонах.
