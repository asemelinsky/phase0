### ЗАВДАННЯ 1.A — Генерація 30 завдань (контент)

**Хто робить:** Олексій + Claude (skill `generate-task`) → Олексій (педагогічний ревью)

**Що потрібно:**
30 JSON-конфігурацій завдань в одному файлі `data/tasks.json` (масив), організованих за 3 рівнями:
- **Новачок** (завдання 1–10): послідовність, базові блоки
- **Середнячок** (завдання 11–20): цикли, умови
- **Досвідчений** (завдання 21–30): комбінації, оптимізація

**Формат JSON одного завдання:**
```json
{
  "id": 1,
  "level": "novachok",
  "title": "Котик і зірка",
  "character": "cat",
  "description": "Зроби так щоб котик пройшов до зірки",

  "availableBlocks": ["event_flag", "motion_forward"],

  "checkConditions": {
    "hasEventBlock": true,
    "hasMotionBlock": true,
    "blocksConnected": ["motion_forward"]
  },

  "hints": {
    "initial": "Привіт! Знайди жовтий блок і перетягни його в робочу область!",
    "success": "Чудово! Котик рухається! Тепер натисни прапорець!",
    "fail": "Майже! Спробуй з'єднати жовтий і синій блоки разом."
  },

  "canvasConfig": {
    "goal": { "x": 500, "y": 320, "emoji": "⭐" },
    "character": { "startX": 50, "startY": 320 }
  }
}
```

**Де зберігати:** `data/tasks.json` — єдиний файл з масивом усіх 30 завдань.

**Критерії прийняття:**
- [ ] 10 завдань рівня «Новачок» готові і перевірені
- [ ] 10 завдань рівня «Середнячок» готові і перевірені
- [ ] 10 завдань рівня «Досвідчений» готові і перевірені
- [ ] Складність росте поступово (1 новий блок за раз)
- [ ] Олексій особисто перевірив кожне завдання з точки зору педагогіки
- [ ] Всі 30 завдань в `data/tasks.json` у вигляді масиву

### ЗАВДАННЯ 1.B — Сторінка завдання `app/task.html?uid=`

**Хто робить:** розробник

**Що потрібно:**
Сторінка `app/task.html` яка:
1. Читає `?uid=abc123` з URL
2. Визначає «завдання дня» на основі дати реєстрації (з localStorage)
3. Завантажує правильне завдання з `data/tasks.json`
4. Показує Blockly-робочу область з потрібними блоками
5. Показує Canvas зі сценою і персонажем
6. Зберігає прогрес у localStorage

**Logіка «завдання дня»:**
```javascript
// Дата реєстрації зберігається при першому відкритті
const registrationDate = localStorage.getItem(`reg_${uid}`) || new Date().toISOString();
const daysSinceReg = Math.floor((Date.now() - new Date(registrationDate)) / 86400000);
const taskIndex = Math.min(daysSinceReg, tasks.length - 1); // завдання 0..29
```

**localStorage структура:**
```javascript
`reg_${uid}`      → ISO дата реєстрації
`progress_${uid}` → { completedTasks: [1,2,3], currentTask: 4, streak: 3 }
```

**UI-елементи (мінімальні):**
- Заголовок: «День X з 30» + назва завдання
- Blockly-робоча область (зліва)
- Canvas зі сценою (справа)
- Кнопки: 🚩 Запустити / ✅ Перевірити / 🔄 Скинути
- Блок підказки знизу
- Прогрес-бар або список пройдених завдань

**Критерії прийняття:**
- [ ] `app/task.html?uid=test123` відкривається і показує завдання
- [ ] Зміна дати в localStorage переключає завдання
- [ ] Blockly показує лише блоки з `availableBlocks` поточного завдання
- [ ] Canvas анімує персонажа при натисканні 🚩
- [ ] Перевірка порівнює workspace з `checkConditions`
- [ ] Прогрес зберігається і відновлюється при повторному відкритті
- [ ] Responsive — працює на планшеті
- [ ] Задеплоєно на Vercel

## 🗓️ СПРИНТ 2 — Telegram-бот + Закритий тест (тижні 3–4)

### ЗАВДАННЯ 2.A — Telegram-бот (реєстрація + розсилка)

**Що потрібно:**
Telegram-бот який:
1. При `/start` — генерує унікальний `uid`, надсилає посилання `site.com/app/task.html?uid=XXX`
2. Щодня о 9:00 (Kyiv time) — надсилає повідомлення з назвою завдання дня + посиланням
3. `/progress` — показує поточну статистику

**Stack:** Node.js (telegraf.js), деплой на Railway/Render (free tier)
**Файли:** `bot/index.js`, `bot/package.json`

**Структура даних бота:**
```json
{
  "chat_id": 123456789,
  "uid": "abc123",
  "registered_at": "2026-03-01T10:00:00Z"
}
```

**Критерії прийняття:**
- [ ] `/start` → генерує uid → надсилає вітальне повідомлення + перше посилання
- [ ] Щоденна розсилка о 9:00 Kyiv time
- [ ] `/progress` → відповідь з кількістю пройдених завдань і серією
- [ ] Бот не падає (є базова обробка помилок)
- [ ] Задеплоєно і постійно онлайн

### ЗАВДАННЯ 2.B — Закритий тест з 20–30 дітьми

- Надіслати посилання на бота батькам учнів Олексія
- Зібрати фідбек через 5–7 днів

**Критерії прийняття:**
- [ ] Мінімум 20 дітей зареєстровані через бота
- [ ] Мінімум 15 з них пройшли перше завдання
- [ ] Фідбек зафіксований у Notion

## 🗓️ СПРИНТ 3 — Бекенд, Прогрес, Retention (тижні 5–6)

### ЗАВДАННЯ 3.A — Supabase: збереження прогресу на сервері

```sql
CREATE TABLE progress (
  uid TEXT NOT NULL,
  task_id INTEGER NOT NULL,
  result TEXT CHECK (result IN ('pass', 'fail')),
  completed_at TIMESTAMPTZ DEFAULT now(),
  PRIMARY KEY (uid, task_id)
);
```

**API виклик з `app/task.html`:**
```javascript
await fetch('https://your-project.supabase.co/rest/v1/progress', {
  method: 'POST',
  headers: { 'apikey': SUPABASE_ANON_KEY, 'Content-Type': 'application/json' },
  body: JSON.stringify({ uid, task_id: currentTask, result: 'pass' })
});
```
