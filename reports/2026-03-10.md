# Звіт — Кодомандри Phase 0
**Дата:** 2026-03-10
**Проєкт:** phase0 — веб-платформа для навчання програмування дітей 6–9 років

---

## Короткий огляд

Сьогодні повністю реалізована система **навчальних анімацій для презентації нових Blockly-блоків** (`intro_block`). Від концепції до working implementation: SVG-рендер реального блоку, словник `BLOCK_DEFS` як єдине джерело правди, автоматичне тегування завдань по сімействах блоків, підключення через `AnimEngine` в `task.js`. Паралельно — рефакторинг `task.html` на окремі файли, створення `animations/` папки з engine-архітектурою, і оновлення всієї skill-інфраструктури.

## Виконані завдання

### 1. Рефакторинг task.html → task.css + task.js
- **Що зробили:** Розбили монолітний `task.html` (2244 рядки) на окремі файли
- **Файли:** `app/task.html` (126 рядків), `app/task.css` (860 рядків), `app/task.js` (1257 рядків)
- **Результат:** HTML містить тільки розмітку, CSS і один `<script src="task.js">`

### 2. AnimEngine — реєстр анімацій
- **Що зробили:** Створили `app/animations/engine.js` — глобальний реєстр `register(id, anim)` / `trigger(id, ctx)`
- **Файли:** `app/animations/engine.js`
- **Результат:** Всі анімації реєструються незалежно, викликаються через єдиний інтерфейс

### 3. Витягнуто існуючі анімації в окремі файли
- **Що зробили:** Перенесли з `task.js` і `progress.html` усі анімації в окремі JS/CSS файли
- **Файли:** `fab_tooltip.js/css`, `onboarding.js/css`, `warmup_dialog.js/css`, `tour.js/css`, `intro_overlay.js/css`
- **Результат:** `progress.html` скорочено з 1159 до ~503 рядків

### 4. preview.html — тестування анімацій без сервера
- **Що зробили:** Standalone-сторінка для перегляду анімацій без запуску сервера
- **Файли:** `app/animations/preview.html`
- **Результат:** Масив `PREVIEWS[]` з кнопками ▶ / ↺ для кожної анімації

### 5. SVG-рендер Blockly-блоку в intro_overlay
- **Що зробили:** `renderBlockSVG(hex, label)` генерує SVG з правильним кольором, notch-конекторами, парсингом `(10)` → input-поле, анімацією `blockDrop`
- **Файли:** `app/animations/intro_overlay.js`, `app/animations/intro_overlay.css`
- **Результат:** Блок у оверлеї виглядає як справжній Blockly-блок

### 6. BLOCK_DEFS — єдине джерело правди для блоків
- **Що зробили:** Словник у `intro_overlay.js` з `label`, `hex`, `desc` для всіх блоків — збігається з Blockly-визначеннями в `task.js`. `intro_block` у tasks.json тепер тільки `{ "id": "..." }`
- **Файли:** `app/animations/intro_overlay.js`, `data/tasks.json`
- **Результат:** Одне місце для змін — немає дублювання між tasks.json і Blockly

### 7. Підключення intro_overlay до task.js через AnimEngine
- **Що зробили:** `task.js` читає `task.intro_block` і викликає `AnimEngine.trigger('intro_overlay', { blockId, uid })`. Ключ localStorage: `intro_block_{blockId}_{uid}` (по блоку, не по завданню)
- **Файли:** `app/task.js`, `app/task.html`
- **Результат:** Анімація показується один раз на блок, а не на завдання

### 8. scripts/tag-intro-blocks.js — автотегування завдань
- **Що зробили:** Скрипт проходить по `tasks.json`, визначає сімейства блоків (`movement`, `pencil`, `loops` і т.д.), проставляє `intro_block` для першого завдання кожного нового сімейства. Також перевіряє `BLOCK_DEFS` і додає плейсхолдери для відсутніх блоків
- **Файли:** `scripts/tag-intro-blocks.js`, `data/tasks.json`
- **Результат:** `task_d1_m1` → `move_right`, `task_d3_m1` → `move_right_steps`, `task_d5_m1` → `pencil` (group)
