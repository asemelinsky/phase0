# Звіт про виконану роботу (Ревізія 2)
**Дата:** 2026-03-06
**Проєкт:** Scratch Learning (Фаза 0) - MVP для перевірки гіпотез

## Короткий огляд
Сьогоднішній день був присвячений завершенню логіки валідації ігрових завдань (Спринт 1), розробці бекенд-логіки Telegram-бота та, найголовніше, повному розгортанню проєкту в інтернеті (завершення Спринту 2). Створено інфраструктуру, готову до старту закритого тестування.

## Деталі виконаних завдань

### 1. Валідація ігрових завдань (Tasks Logic)
- **Проблема:** Завдання рівня "Pro", які використовували безкінечний цикл `always`, не збігалися з цільовими координатами при симуляції та викликали помилки валідації.
- **Рішення:** Написано та відлагоджено скрипт `scripts/validate-tasks.js`, що підтримує команди `--check` та `--fix`.
- **Результат:** Всі 30 ігрових завдань (`data/tasks.json`) успішно пройшли 100% валідацію.

### 2. Створення Telegram Бота (Node.js)
- **Проблема:** Необхідність механізму реєстрації, сповіщень та retention.
- **Рішення:** 
  - Реалізовано Node.js бота за допомогою `telegraf`.
  - Команда `/start` генерує `uid` та персональне посилання на "Мапу пригод".
  - Команда `/progress` обчислює кількість відкритих завдань (по днях).
  - Налаштовано `node-cron` на розсилку завдань дня всім дітям о 9:00 (Київ).

### 3. Додаткові компоненти (DoD Спринту 1)
- **Blockly + Canvas + Підказки:** У файлі `app/task.html` інтегровано робочу версію Blockly, відмальований Canvas з ігровою логікою та блоком підказок (Hints), які змінюються в залежності від успіху чи помилки.
- **Responsive (мобільна адаптивність):** ⚠️ *Уточнення:* На даний момент в CSS не прописані `@media`-запити. Фронтенд наразі розрахований переважно на десктоп/дефолтний розмір. Це потребує покращення (Спринт 3 або окремий фікс) перед масовим запрошенням користувачів на смартфонах.

### 4. Забезпечення надійності Бота (Fallback Data Storage)
- **Проблема:** Під час обговорення виявлено ризик втрати In-Memory масиву користувачів під час будь-якого перезапуску сервера бота. Це могло зруйнувати закритий тест.
- **Рішення:** Написано логіку читання/запису списку підписників у файл `bot/data/users.json` при кожній реєстрації. 
- **Результат:** Бот тепер запам'ятовує користувачів між рестартами. Папку `bot/data/` додано до `.gitignore`, щоб убезпечити персональні ідентифікатори (Telegram ID).

### 4. Деплой Frontend-частини на Vercel (Спринт 1 - DoD)
- **Проблема:** Бот генерував посилання на `localhost:3000`, які неможливо надати реальним користувачам.
- **Рішення:**
  - Створено файл `vercel.json`.
  - Проєкт автоматично розгорнуто на Vercel через прив'язку GitHub-репозиторію.
  - Посилання `localhost:3000` в коді бота замінено на отриманий Production URL (`https://phase0-five.vercel.app`).
- **Результат:** Спринт 1 офіційно виконано. Клієнтський застосунок доступний в інтернеті.

### 5. Деплой Telegram-бота на Render (Спринт 2 - DoD)
- **Проблема:** Бот має безперервно працювати 24/7 (як Background Worker), відстежуючи розклад `cron`. При спробі безкоштовного деплою як Web Service, Render завершував збірку через "Port scan timeout".
- **Рішення:**
  - Прописано Root Directory: `bot` та змінні оточення (включаючи `BOT_TOKEN`).
   - До коду бота (`index.js`) додано фіктивний HTTP-сервер (модуль `http`), який слухає `process.env.PORT`. Це слугує Health-check'ом для Render і підтверджує, що сервіс активний.
- **Результат:** Деплой успішний (`phase0 is live!`). **Спринт 2 закрито повністю.**

## Наступні кроки (Спринт 3+)
На даний момент проєкт вважається технічно готовим до запуску закритого бета-тестування на поточній інфраструктурі (Vercel + Render + Local JSON Base), **проте з урахуванням того, що мобільна версія (Responsive) ще не оптимізована**.

Наступним після збору першого фідбеку буде перехід до **Спринту 3**:
1. Адаптація Frontend-у (Responsive дизайн для смартфонів та планшетів).
2. Переніс зберігання даних з JSON-файлу у хмарну БД **Supabase**.
3. Збереження не тільки факту реєстрації (uid/chatId) у БД, а й конкретних подій в грі (наприклад, скільки разів дитина натиснула перевірку на поточному кроці).
