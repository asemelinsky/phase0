# Звіт про виконану роботу
**Дата:** 2026-03-09
**Проєкт:** Кодомандри (Фаза 0) — MVP для перевірки гіпотез

---

## Короткий огляд

Сесія зосереджена на двох великих напрямках:
1. **Система малювання (Days 5–8)** — повна реалізація олівцевих завдань: блоки pencil_down/up, валідація форм, drawing_guide контури, режими задач.
2. **challenge_d7 (КОД)** — перероблено з нуля: drawing_shape режим, penZones валідація, drawing_guide літер у межах canvas, admin panel для тестування.

16 комітів за сесію. Усі задеплоєно на Vercel.

## Деталі виконаних завдань

### 1. Олівцеві блоки і координати (Days 5–8)
- **Проблема:** `pencil_down`/`pencil_up` блоки не були реалізовані в рушії. Координати завдань 5–8 виходили за межі сцени.
- **Рішення:**
  - `e4e8746` — реалізовано `pencil_down`/`pencil_up` у `executeChain`: `pencilDown` flag, `pencilTrail` масив, рендеринг лінії на canvas.
  - `e08e460` — автокорекція координат для Days 5–8 через `validate-tasks.js --fix`.
- **Результат:** Герой малює лінії при опущеному олівці на Days 5–8.

### 2. Система валідації малюнку (drawingPattern)
- **Проблема:** Не було способу перевірити чи дитина намалювала правильну форму (сходинки, замкнену фігуру, тощо).
- **Рішення (`3d12006`):**
  - `validateDrawing(pattern, log)` — нова функція валідації.
  - `getPenSegments(log)` — розбиває moveLog на окремі pen-down сегменти.
  - `isClosedShape(seg)` — перевіряє замкнення (старт = фінал).
  - `isStairs(dirs, count)` — перевіряє сходинки (чергування двох напрямків).
  - Підтримка полів: `requirePen`, `figureType` (stairs/closedShape), `stairCount`, `figures`, `requireGap`.
- **Результат:** Tasks 5–7 мають реальну перевірку форми намальованого.

### 3. Фікс isStairs для нерівних сходинок
- **Проблема:** `isStairs` відхиляв коректне рішення `right(3),up(2),right(3),up(2)` — вимагав щоб всі сегменти були однакової довжини.
- **Рішення (`3eddfa5`):** Перевірка змінена — горизонтальні сегменти мають бути однаковими між собою, вертикальні — між собою, але ці дві групи можуть відрізнятися.
- **Результат:** Сходинки з різними кроками по X і Y тепер проходять валідацію.

### 4. Три режими завдань: navigation / drawing_navigation / drawing_shape
- **Проблема:** Рушій не розрізняв задачі "дійди до зірки" від "намалюй фігуру" — це призводило до конфліктів (виконання зупинялось на цілі посеред малювання).
- **Рішення:**
  - `8735547` — early-exit (зупинка при досягненні цілі) пропускається якщо `drawingPattern.figureType` задано.
  - `c2b2ce0` — success logic: якщо `figureType || target_type === 'none'` → успіх базується на малюнку, не позиції.
  - `8964ccb` — оновлено skill `generate-task`: документовані три режими з правилами для кожного.
- **Результат:** challenge_d6 (два квадрати) більше не зупиняється посеред малювання.

### 5. drawing_guide — контури-підказки на сцені
- **Проблема:** Для завдань з літерами дитина не знає яку форму малювати.
- **Рішення (`3dfbb05`):**
  - Поле `drawing_guide` в tasks.json: масив штрихів `[[x,y],[x,y],...]`.
  - Рендеринг: блідо-фіолетові лінії (`rgba(180,100,220,0.22)`) під pencil trail, з offset +25px.
  - Оновлено skill `generate-task`: правила координат (x≤550, y≤400), тільки прямі кути, приклад КОД.
- **Результат:** challenge_d7 показує контури літер К, О, Д як підказку.

### 6. challenge_d7 — повне перероблення
- **Проблема:** challenge_d7 мав `target_type:"star"` з координатами x=850 (поза canvas 600px), drawing_guide з координатами до x=800, невірний drawingPattern.
- **Рішення (`647571c`, `26fb21b`):**
  - `target_type:"none"` — зірка не відображається, позиція не перевіряється.
  - `drawingPattern: {requirePen, figures:3, requireGap}`.
  - Нова `drawing_guide`: К (x=50–150), О (x=200–350), Д (x=400–550 з ніжками y=400) — всі в межах canvas.
  - Новий `solution`: три окремих прямокутних штрихи в межах координат.
- **Результат:** challenge_d7 коректно рендериться і завершується успіхом.

### 7. penZones валідація
- **Проблема:** Дитина могла намалювати 3 штрихи в одному куті і отримати успіх для завдання "намалюй КОД".
- **Рішення (`448c699`, `6fde3f1`):**
  - Нове поле `penZones` у drawingPattern: масив зон з `xMin/xMax/msg`.
  - `validateDrawing` перевіряє: в кожній зоні є хоча б один рух з `penDown=true`.
  - Додано `x, y` до всіх `moveLog.push` записів (раніше були тільки `dir` і `penDown`).
  - challenge_d7 отримав `penZones` для зон К/О/Д.
  - Оновлено skill `generate-task` — документація `penZones`.
- **Результат:** Валідація перевіряє педагогічну ціль: олівець вниз у правильному місці.

### 8. Workspace auto-save + Admin panel
- **Проблема:** При тестуванні challenge_d7 потрібно щоразу вручну набирати 30+ блоків.
- **Рішення (`6097422`, `f1d005b`):**
  - Auto-save: workspace зберігається в localStorage при кожній зміні (ключ: `ws_${taskId}`).
  - Auto-load: при відкритті задачі workspace відновлюється.
  - `?admin` URL параметр показує панель: 📋 Копіювати XML / 📥 Вставити XML / 🗑️ Скинути.
  - Фікс: `Blockly.Xml.textToDom` замінено на `DOMParser` (видалено в Blockly v9+).
- **Результат:** Тестування олівцевих завдань стало в рази швидшим.
